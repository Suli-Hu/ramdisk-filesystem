1 ramdisk-filesystem
2 ==================
3 
4 Custom file system created for the linux 2.6 kernel. Final project for CS552.
5 
6 Consists of a kernel loadable module with an interface into a library, RAMFileLib. The
7 module initializes, a set amount of memory in the kernel, and creates a data structure
8 to manage blocks of data.
9 
10 The RAMFileLib interfaces with the module using IOCTL calls. Each process which uses
11 the library creates their own process file handle block, although this was not the focus of
12 this project.
13 
14 The file system provides support for creating files, creating directories, reading files from
15 a directory, writing data to files, and reading data from files. All of the functions provide
16 clear error conditions.
17 
18 Running
19 ==================
20 
21 To work with the filesystem, run the remake.sh script as root. This creates and builds the filesystem, loads it into memory, and also builds the test_file for testing. If you want to create your own test file, or work with the filesystem in any other way, in any file that
22 uses RAMFileLib, the following extern calls must be included:
23 
24  extern int proc;
25  extern int currentFdNum;
26  extern std::vector<FD_entry> fd_Table;
27 
28 This is required for the library to work correctly. We intended to remove this dependency but it was beyond the time we had available.
29 
30 To test the filesystem, simply run
31 
32  ./user
33 
34 and the test script will run and verify that the filesystem is working correctly. Different tests can be turned on or off within test_file.cpp
